# 汽车销售管理系统 - 开发对话记录

## 第一阶段：初始搭建

### 项目初始化
- 创建 Spring Boot 后端项目结构（Maven + JPA）
- 创建 Vue 3 + Vite 客户端项目
- 创建 Vue 3 + Vite + ECharts 管理端项目
- 配置前后端代理、跨域、数据库连接

### 数据库设计
- 设计表结构：customer、car、appointment、purchase_order
- 3 个触发器：确认订单自动扣库存、取消订单自动恢复库存、库存为0自动停售
- schema.sql + data.sql（8种车型、3个客户、5条预约、5条订单示例数据）

### 后端开发
- 实体类 → Repository → Service → Controller
- REST API：车辆CRUD、客户查询、预约创建/确认/取消、订单创建/确认/取消
- 统计分析 API：销量排行、销售额占比、价格区间统计
- Excel 批量导入 + 模板下载

### 前端开发
- 客户端：车辆列表（含筛选器）、车辆详情、预约试驾、提交订单、查询中心
- 管理端：仪表盘概览、车辆管理（含 Excel 导入）、客户管理、预约管理、订单管理、综合查询、统计分析（ECharts 图表）

## 第三阶段：交付工具

### 一键启动/关闭脚本
- start.bat：编译后端 + 启动后端 + 启动前端 + 验证 API
- stop-all.ps1 / stop.bat：按端口杀进程
- 启动倒计时 + 端口轮询检查
- 自动打开浏览器

### 独立 exe 工具
- **启动系统.exe**：一键启动所有服务 + 倒计时 + 自动打开浏览器（.NET 8 PublishSingleFile）
- **关闭系统.exe**：一键停止所有服务
- **配置检查.exe**：11项环境检查
  1. JDK版本
  2. Node.js + npm
  3. 后端 Jar（缺失可自动编译）
  4. 前端 node_modules（缺失可自动安装）
  5. 端口占用（8080/5173/3000）
  6. MySQL 连接（提示输入账号密码）
  7. 数据库 car_sales_db（自动创建）
  8. 数据表结构（从 schema.sql 建表）
  9. 种子数据（从 data.sql 插入）
  10. application.yml 配置更新
  11. 前端构建状态
