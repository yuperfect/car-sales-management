# Sprint Plan

## Overview

该项目采用单次交付模式（课程设计），没有多轮 sprint。所有工作在一个连续周期内完成。

| 阶段 | 时间段 | 主要工作 |
|------|--------|---------|
| 设计 | 前期 | 需求分析、HLD、LLD、数据库设计、产品Backlog |
| 后端开发 | 中期 | 实体类→Repository→Service→Controller，完整的 REST API |
| 前端开发 | 中期 | 客户端(Vue3) + 管理端(Vue3+ECharts) |
| 联调测试 | 后期 | API 联调、Bug 修复、E2E 测试 |
| 交付 | 末期 | 一键启停脚本、配置检查工具、README |

## 已实现功能

### 客户端（用户端）
- 在售车辆列表浏览 + 筛选（品牌/型号/价格区间）
- 车辆详情页（完整参数与价格）
- 预约试驾（姓名+电话+车辆+时间）
- 按编号查询预约状态
- 提交购车订单（姓名+电话+车辆+数量）
- 按编号查询订单状态
- 统一查询中心

### 管理端（管理员端）
- 仪表盘概览（车辆/客户/待处理数量）
- 车辆管理（增删改 + Excel批量导入 + 模板下载）
- 客户管理（列表 + 关键词搜索）
- 预约管理（确认/取消）
- 订单管理（确认/取消，确认后自动扣库存）
- 综合查询（按车型/价格/客户/日期筛选）
- 统计分析（销量排行柱状图、价格区间饼图、销售额占比）

## 构建工具
- 启动系统.exe（一键启动 + 倒计时 + 自动打开浏览器）
- 关闭系统.exe（一键停止所有服务）
- 配置检查.exe（11项环境检查 + 数据库初始化）
